面向对象的阶段概念：    OOA：面向对象分析    OOD：面向对象设计    OOP：面向对象编程
面向对象的三大特征：封装，继承，多态
PHP面向对象：
    1、类是抽象的，概念的，代表一类事物，比如人类，猫类。
    2、对象是具体的，实际的，代表一个具体事物。
    3、类是对象的模版，对象由类实例化出来的。

class 类的三种属性：   属性（变量）  方法（函数）  类常量（常量）

类有5种成员： 非静态属性、静态属性、非静态方法、静态方法、类常量

类的定义：
    1，定义类的时候，类名的后面不要有小括号
    2，需要对成员（属性和方法）声明一下才可以使用！
    3，定义属性的时候，可以给该属性一个默认值，但是该默认值一定是一个“直接值”，而不是通过计算得到的值！

实例化对象：      $obj = new class();
访问对象方法：    $obj -> fun();
访问对象属性：    $obj -> name;
类中访问本身的方法：  $this->fun();

所有常量定义后不能在修改
常量：常量名应该全部大写，并且前面不要有 $
    基本语法 ： const 常量名 = 值;
    访问：  类名::常量名; 或者  接口名::常量名  或者 self::常量名;

构造方法：对象的初始化，创建对象自动运行
    public function __construct( 参数 ){
        //初始化操作
    }

析构方法: 对象“消失”的时候由系统自动调用的 
    public function __destruct( 参数 ){
        //初始化操作
    }

global 变量名  ：定义全局变量

静态变量：所有对象都共享的变量
    public static 变量名   ：类的内部定义
    self::变量名           ：在类的内部访问
    类名::变量名           ：在类的外部访问，，注意：是类名

***静态方法只能操作静态变量，非静态方法可以操作静态变量
静态方法：所有对象都共享的方法
    public static 方法名    ：在类的内部定义
    self::方法名            ：在类的内部访问
    类名::方法名            :在类的外部访问，，注意：是类名


封装：
    访问控制修饰符（public、protected、private）
        public      ：公开的，可以在类的内部，外部和子类都可以访问
        protected   ：受保护的，只有在类和子类中可以访问
        private     ：私有的，只有在本类内部可以访问
        涉及到俩个不常用的魔术方法 __set() 和 __get()  

继承：可以多层继承
    class 类名 extends 父类( ){  }
    php在继承时，默认是不自动调用父类的方法（包括构造方法），想要自动调用，在类的构造方法里，这么写：  父类名::父类的方法名  可以是构造方法

多态：
    重载：是类多态的一种实现
    重载概念，函数名一样，通过函数的参数个数或者是参数类型不同，达到调用同一个函数名，但可以区分不同的函数
    重载涉及到一个魔术方法 __call( 方法名，参数 )

抽象类：
    1、本身不需要实例化，用来让其他类去继承他
    2、用abstract 关键字修饰一个类，这个类就是抽象类，用abstract 关键字修饰一个方法，这个方法就是抽象方法（抽象方法就不能有方法体）；
    3、抽象类中可以没有抽象方法，但是一但有抽象方法，类就必须是抽象类
    4、如果一个类继承了抽象类，就必须实现抽象类中的所有抽象方法，除非自己也是抽象类

    基本语法：
        abstract class 类名{
            abstract 访问修饰符 function 函数名();
        }

接口：
    接口的作用就是：声明一些方法，提供类来实现，接口还体现编程中我们希望的效果，高内聚低藕合的特点
    1、接口不能实例化
    2、接口中的方法不能有方法体
    3、一个类中可以实现多个接口。用逗号隔开
    4、接口中可以有属性，但必须是常量，默认是public
    5、接口中的方法都必须是public ，默认就是public
    6、一个接口不能继承其他类，但是可以继承别的接口
    7、当一个类去实现某个接口时，这个类就必须实现接口中的所有方法

    在什么情况下可以考虑使用接口：
        1、定规范   2、定下规范让别的程序员实现  3、当多个类，实现一类操作，操作的方式不同的时候

    定义接口：
        interface 接口名 {
            public function start();
            public function stop();
        }

    实现接口类：
        class 类名 implements 接口名1，接口名2... {
            public function start(){
                echo '接口开始';
            };
            public function stop(){
                echo '接口结束';                
            };
        }

final : 
    不被任何类继承基本语法：
        final class 类名{   }
    父类的方法，不希望子类改写：
        父类这么写：
        class 类名{  
            final public function start(){
            
            };
        }


有一个常见的删除一个对象的方法：$对象变量 = NULL;
销毁对象 ： unset(对象);
