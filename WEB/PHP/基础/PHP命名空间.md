    PHP是在php5以后中才开始使用！ ThinkPHP3.2版本以后就使用了命名空间！
    简单来说，命名空间就是给不同的内存空间起一个不同的名字！

    命名空间的定义:
        使用关键字：namespace  语法： namespace 空间名字;

    需要几个注意的地方：
        1，  如果一个脚本的开始需要定义命名空间，则必须在脚本的最开始处定义！
        2，  一个脚本周期内，可以定义多个命名空间，并且不同的空间可以定义相同名字的函数、常量和类！
        3,   变量不属于空间成员！

    空间成员:    class     function    const
        一个命名空间内可以有任何的PHP代码！但是命名空间本身只“管辖”三种空间成员：常量、函数和类，其他的都不属于空间成员！
        空间成员（类、函数和常量）受命名空间的限制之外，其他的代码都是该怎么执行就怎么执行，跟没有命名空间一样不存在任何的差别！

    子空间:
        同计算机的目录可以存在子目录一样，内存中的空间也可以存在子空间！
        空间：namespace A\B\C，其中C就是B的子空间，而B又是A的子空间，注意要是要反斜杠\，其表示的含义是创建了一个名字为C的子空间，并且A和B空间也一起创建了！

    空间成员的访问:
        PHP提供了三种访问空间成员的方式：
        非限定名称访问
        限定名称访问
        完全限定名称访问


        非限定名称访问:   就是在访问空间成员的时候，没有指定具体的空间的名字！
        限定名称访问:     就是从当前空间开始访问其子空间的成员！
            语法形式为：子空间名\空间成员名
        完全限定名称访问:  就是直接从根空间（类比根目录）开始对绝对路径访问！
            语法形式为：\空间1\空间2\……\空间成员，其中这里的\代表根空间！

    采用完全限定名称访问的最大好处是可以在任意的空间访问其他任意空间的成员！

    空间引入: 采用完全限定名称
        当一个脚本包含了另外一个脚本的时候，就出现了空间引入的问题！
    
    空间类的引入:空间类的引入就是把其他空间的类，引入到当前空间，在当前空间直接使用这个类！目的就是可以提高代码执行的效率！
        use 空间名\类名
    当其他空间引人的类于本空间的类名字一样时，我们就需要将被引入的那个类起一个别名： as 别名；


    全局空间：不给脚本定义命名空间，脚本中所定义的内容就属于全局空间！
